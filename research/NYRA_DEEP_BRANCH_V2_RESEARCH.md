# Nyra Deep Branch Architecture V2 — Research dossier

## Scope, source of truth and decision rule

This dossier analyses the catalogue returned by the authenticated `codexai` Render request
`GET /v1/nira/branches` captured on 2026-07-23. It contains 18 branches and 239 subbranches.
Every live subbranch is represented below exactly once. The repository V1 catalogue was used only
as a read-only equivalence check; the authenticated live response remains the source of truth.

The research rule is deliberately restrictive: a proposed specialized capability is admissible only
when it has a distinct decision or transformation, observable inputs and outputs, evidence that can
be checked, a bounded failure surface, and a clear boundary from neighbouring nodes. Names in
`candidate capabilities` describe actual mechanisms, not folders. Architecture may split a candidate
into method, strategy, verifier and metric nodes only where each child remains independently useful.
Universal Core remains the sole policy, release and execution authority.

Column semantics:

- **Problem / use case** states the concrete problem and representative workload.
- **Failure modes** identifies observable ways the node can be wrong or unsafe.
- **Required evidence** is the minimum evidence needed before confidence can be assigned.
- **Candidate capabilities** are non-generic Level-2 mechanisms recommended for contract design.
- **Boundary / overlap** states ownership and prevents semantic duplication.
